# UI 与后端第一阶段对齐设计

## 范围

只修改 `ui/`、`app/styles.py`、`tests/test_ui/`，并新增一份接口 issue 文档。
不修改 `core/`、`store/`、`app/app.py`、依赖或其他成员文件。

## 已具备接口并立即接入

### 订阅同步

侧栏同步图标点击后打开紧凑菜单，包含“同步当前订阅”和“同步全部订阅”。没有选中
订阅时禁用前者，但仍允许同步全部。批量同步沿用 `SyncService.sync_all()` 与现有信号。

### 订阅源管理

订阅列表增加右键菜单，提供重命名与删除。重命名使用单行输入框并保留原名称；删除在
主窗口显示二次确认。UI 只发出意图，MainWindow 复用已经注入的 `FeedStore.update()`
和 `FeedStore.delete()`。删除当前订阅后清空文章和 Reader，避免残留无效选择。

### 搜索范围

文章搜索框与一个窄型范围选择器同行显示。范围为“当前”和“全部”，默认当前。切换
范围后若搜索框已有文字则立即刷新；全局搜索调用现有 `EntryStore.search(query,
feed_id=None)`。文章栏标题和搜索占位文字同步表达当前范围。

## 后续布局预留

不放置无法工作的占位按钮。后续接口齐备后采用以下结构：

- Reader 正文下方：可拖动的“AI 摘要 / 笔记”工作区。
- Reader 标题下方：可换行标签徽章和“建议标签”入口。
- 批量工具栏：增加批量打标和 Digest 导出。
- 设置窗口：增加“用量统计”页。
- 侧栏操作菜单：增加 OPML 导入/导出。

缺失接口、建议签名和验收条件写入独立 issue 文档，供对应成员实现。

## 体验修复补充

- 搜索输入框使用精简中文右键菜单，仅保留复制和粘贴。
- 订阅源重命名与删除仅放在列表右键菜单中，删除仍需确认，不增加顶部按钮。
- Reader 的窄、中、宽档位改为响应式宽度。专注模式或 Reader 拉宽时正文居中扩展，
  图片填充正文宽度并保持比例；小窗口继续自动收缩。
- 文章批量栏只显示已选数量和退出按钮；已读、未读、收藏、删除统一放入选中文章的
  右键菜单。订阅源删除也只保留在订阅列表右键菜单中。
