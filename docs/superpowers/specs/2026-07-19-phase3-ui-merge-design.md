# Phase 3 UI 整合设计

## 目标

以 `2026-summer-semester-groupproject-reader-main (4)(1).zip` 为主项目，将
`mercury-reader-phase3-ui-cleanup-complete (8).zip` 中已完成的 UI 功能整合进去，
同时保留主项目更新后的后端、存储、依赖、构建配置和文档。

## 整合范围

- 移植第二个项目中 `ui/` 下的界面改动，并合入 `app/styles.py` 的 UI 样式。
- 移植与界面行为直接对应的 UI 测试。
- 保留主项目独有的摘要、翻译、设置、Digest、NoteStore 和其他后端能力。
- 不复制第二个项目中 `core/`、`store/`、`tools/`、`pyproject.toml`、`uv.lock`、
  `AGENTS.md` 等不属于此次 UI 整合范围的文件。

## 目标功能

- 文章批量选择、批量标记已读/未读、收藏和删除。
- 侧栏、文章栏和 Reader 工具栏使用统一的紧凑方形图标。
- 所有纯图标按钮保留悬浮名称和可访问名称。
- AI 摘要区域可通过分隔条折叠、展开和调整高度。
- 三栏顶部控件对齐。
- 精简翻译、主题、字号、正文宽度、阅读模式和聚焦模式控件。
- 保留隐藏侧栏、恢复侧栏、Reader 聚焦/全屏阅读等既有功能。

## 兼容策略

先比较每个目标文件与主项目的接口差异，再进行选择性合并；若第二个项目引用主项目
不存在或签名不同的接口，以主项目当前接口为准进行适配。禁止通过整体覆盖主项目来
解决冲突。

## 验证

- UI 整合测试先失败、整合后通过。
- 运行 Python 编译检查和 Ruff UI 范围检查。
- 在可用环境中运行 pytest；若 Qt 系统库缺失，区分环境问题与代码问题并明确报告。
- 使用离屏 Qt 启动主窗口，验证窗口能够创建且主要控件存在。
- 打包时排除虚拟环境、缓存、数据库和临时文件。

## 交付物

提供一个以主项目为基础的整合版 ZIP，并给出 Windows PowerShell 下使用 `uv` 的推荐
运行方式以及不使用 `uv` 的备用运行方式。
