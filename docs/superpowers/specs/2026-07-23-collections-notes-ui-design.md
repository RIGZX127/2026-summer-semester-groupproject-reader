# 收藏夹与笔记 UI 设计

## 范围

本次只接入后端已经提供的 `CollectionStore` 与 `NoteStore`，不修改数据库结构、Store 实现、Digest 导出或标签功能。

## 收藏夹

- 左侧栏在订阅源列表下方增加“收藏夹”区域。
- 收藏夹列表支持选择、新建，以及通过右键菜单重命名和删除。
- 选择收藏夹后，中间文章列表显示该收藏夹内的文章。
- 文章右键菜单增加“添加到收藏夹”请求；MainWindow 展示收藏夹选择对话框并调用 `CollectionStore.add_entry()`。
- 旧的星标继续使用 `EntryStore.toggle_star()`，同时通过 `CollectionStore.quick_star()` / `quick_unstar()` 同步默认收藏夹，避免两套收藏状态继续分离。
- 删除收藏夹必须确认；默认收藏夹允许重命名，但删除后可由下一次快捷收藏重新创建。

## 笔记

- Reader 下方使用标签页容纳现有 AI 摘要与新增笔记编辑器。
- 笔记编辑器显示 Markdown 文本、占位提示和“未保存 / 正在保存 / 已保存 / 保存失败”状态。
- 切换文章时停止旧计时器并加载新文章的笔记；无文章时禁用编辑。
- 用户编辑后等待 5 秒发出保存请求；切换文章和关闭窗口前立即保存尚未保存的内容。
- MainWindow 负责调用 `NoteStore.get()` / `save()`，并使用 entry_id 校验异步返回，避免旧文章结果覆盖当前文章。

## 错误与兼容

- 未注入相应 Store 时隐藏收藏夹入口、禁用笔记编辑，不影响现有阅读功能和测试。
- Store 异常通过状态栏或笔记编辑器状态显示，不静默吞错。
- 所有新图标按钮提供 Tooltip 与可访问名称；用户可见字符串使用 Qt 翻译入口。
