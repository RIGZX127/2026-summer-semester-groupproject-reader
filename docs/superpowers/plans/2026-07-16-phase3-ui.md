# Phase 3 UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the documented Phase 3 settings, summary, and translation UI without moving business logic into the UI layer.

**Architecture:** Qt widgets emit user intent and render typed `AgentUIEvent` values from the existing shared runtime. `QSettings` owns UI preferences, while provider testing remains an injected asynchronous action rather than a network call inside a widget.

**Tech Stack:** Python 3.11, PySide6, pytest, pytest-qt, QWebEngineView.

## Global Constraints

- Only UI-owned files and UI tests are changed, except passing the selected `entry_id` through existing UI orchestration.
- Agent events are accepted only when both `entry_id` and active `run_id` match.
- User-visible strings use `tr()` and controls expose accessible names/tooltips.
- The existing semantic palette and centralized QSS remain the visual source of truth.
